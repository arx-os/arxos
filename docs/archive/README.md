## üìÇ Archive Organization Principles

**Why documents are archived:**
1. **Superseded** - Newer consolidated version exists
2. **Historical** - Captures point-in-time state
3. **Milestone** - Marks completion of specific work
4. **Reference** - May be useful for understanding evolution

**Naming Convention:**
- **Dated files:** `document-name-oct-2025.md` (superseded on that date)
- **Session files:** `SESSION_SUMMARY_OCT_12_2025.md` (created that date)
- **Original files:** Preserved with original capitalization

**What's NOT archived:**
- Current active documentation (see [Documentation Index](../DOCUMENTATION_INDEX.md))
- Code files (these remain in version control)
- Configuration files (remain in `/configs/`)

**Adding to Archive:**

3. Update this README with new category if needed
