# Contributing

Thank you for your interest in improving the Arxos platform!
Please follow the guidelines below to ensure a smooth contribution process.

---

## 1. Code of Conduct

- Be respectful and constructive in all interactions.
- Follow organization and project-specific guidelines.

---

## 2. How to Contribute

### Reporting Bugs

- Search [issues](https://github.com/<org>/arx-docs/issues) to see if your bug is already reported.
- If not, create a new issue with clear steps to reproduce.

### Suggesting Enhancements

- Open an issue describing your enhancement.
- Propose clear use cases and, if possible, a suggested implementation.

### Pull Requests

1. Fork the relevant repo and create a feature branch.
2. Write clear, concise commits.
3. Add or update documentation and tests as necessary.
4. Submit a pull request (PR) and fill out the PR template.
5. Reference related issues in your PR description.

---

## 3. Coding Standards

- Follow language-specific style guides (Go, Python, Swift).
- Write descriptive commit messages.
- Maintain or improve test coverage.
- Format and lint your code before submitting.

---

## 4. Review Process

- PRs require review and approval by at least one core maintainer.
- Address feedback promptly.
- No direct commits to `main` or `production` branches.

---

## 5. Documentation

- Update [arx-docs](https://github.com/<org>/arx-docs) when conventions, APIs, or object models change.
- Link to or update `BUILDING_SYSTEMS_LIBRARY.md` and `API_REFERENCE.md` when relevant.
