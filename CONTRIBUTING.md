### 3. Getting Started

1. **Fork the repository**
2. **Create a feature branch**
   ```bash
   git checkout -b feature/amazing-feature
   ```
3. **Make your changes**
4. **Write/update tests**
5. **Update documentation**
6. **Submit a pull request**

### 4. Code Style

#### Rust
- Use `rustfmt` for formatting
- Follow Rust API guidelines
- Write documentation comments
- Include unit tests

#### Hardware
- Use KiCad for PCB design
- Follow open hardware practices
- Include bill of materials
- Document assembly steps

### 5. Commit Messages

Follow conventional commits:
```
feat: Add LoRa packet routing
fix: Correct ArxObject byte ordering
docs: Update mesh network guide
test: Add protocol unit tests
```

### 6. Pull Request Process

1. **Update documentation** for any API changes
2. **Add tests** for new functionality
3. **Ensure CI passes** (when we add it)
4. **Get review** from maintainers

### 7. Community Guidelines

- **Be respectful** - We're all here to democratize building intelligence
- **Be helpful** - Share knowledge, especially with newcomers
- **Be patient** - This is a revolutionary project, things take time
- **Have fun** - We're literally gamifying building management!

### 8. BILT Token Rewards

Contributors earn BILT tokens (when implemented):
- Bug fix: 100 BILT
- New feature: 500 BILT
- Hardware design: 1000 BILT
- Major feature: 5000 BILT
