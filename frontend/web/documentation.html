Get Started in Under 10 Minutes

Follow this guide to set up your first Arxos building optimization project. You'll learn how to deploy the platform, authenticate, and run your first optimization.

Prerequisites

  • Docker 20.0+ and Docker Compose
  • 8GB RAM minimum (16GB recommended)
  • PostgreSQL 13+ (or use Docker)
  • Redis 6+ (or use Docker)

Step 1: Clone and Deploy

# Clone the repository
git clone https://github.com/arxos/arxos-platform.git
cd arxos-platform

# Start with Docker Compose
docker-compose up -d

# Verify deployment
curl http://localhost:8000/health

Step 2: Access the Platform

Open your browser and navigate to http://localhost:8000

‚úÖ
Success! You should see the Arxos platform landing page with live optimization metrics.

Installation

Docker Installation

    volumes:
      - postgres_data:/var/lib/postgresql/data

    image: redis:6-alpine
    volumes:
      - redis_data:/data

volumes:
  postgres_data:
  redis_data:

Authentication

Arxos uses JWT (JSON Web Token) authentication with role-based access control (RBAC).

Login and Get Token

curl -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin@arxos.com",
    "password": "your-password"
  }'

JavaScript SDK

// Initialize API client
const arxosAPI = new ArxosAPIClient({
    baseURL: 'http://localhost:8000/api/v1'
});

// Authenticate

Running Optimization

// Start genetic algorithm optimization
const optimizationResponse = await arxosAPI.runOptimization({
    building_id: "building_123",
    optimization_config: geneticAlgorithmConfig
});

// Monitor progress via WebSocket
realTimeData.subscribe('optimization', (data) => {
    console.log(`Generation ${data.generation}: Best fitness = ${data.best_fitness}`);
    console.log(`Population diversity: ${data.diversity}`);
    console.log(`Convergence rate: ${data.convergence_rate}%`);
});

// Get final results
const results = await arxosAPI.getOptimizationResults(optimizationResponse.optimization_id);

Docker Deployment

Arxos provides production-ready Docker containers with multi-stage builds for optimal performance and security.

Production Docker Compose

# docker-compose.prod.yml
version: '3.8'

services:
  arxos-api:
    image: arxos/platform:latest
    build:
      context: .
      target: production
    ports:
      - "8000:8000"
    environment:
      - NODE_ENV=production
      - DATABASE_URL=${DATABASE_URL}
      - REDIS_URL=${REDIS_URL}
      - JWT_SECRET=${JWT_SECRET}
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
    deploy:
      replicas: 3
      resources:
        limits:
          memory: 2G
        reservations:
          memory: 1G
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"
        
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
      - ./ssl:/etc/nginx/ssl:ro
    depends_on:
      - arxos-api

  postgres:
    image: postgres:13
    environment:
      - POSTGRES_DB=${POSTGRES_DB}
      - POSTGRES_USER=${POSTGRES_USER}
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD}
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ./backups:/backups
    command: >
      postgres
      -c max_connections=200
      -c shared_buffers=256MB
      -c effective_cache_size=1GB

volumes:
  postgres_data:

Troubleshooting

Common Issues

üö´ Optimization Fails to Converge

Symptoms: Genetic algorithm runs but doesn't improve fitness over generations.

Solutions:
  • Increase population size (try 200+ for complex problems)
  • Adjust mutation rate (0.01-0.05 typically works well)
  • Enable multi-island evolution with migration
  • Check constraint violations - overly restrictive constraints prevent convergence

‚ö†Ô∏è High Memory Usage

Symptoms: System runs out of memory during large optimizations.

Solutions:
  • Reduce population size and increase generations
  • Enable memory pooling in configuration
  • Use streaming optimization for very large problems
  • Scale horizontally with Kubernetes
